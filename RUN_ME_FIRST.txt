WHAT TO EXPECT:

1. Backend starts and says "Uvicorn running on http://0.0.0.0:8000"

2. On first request, it will:
   - Download GPT2 model (500MB, one time only)
   - Load the model into memory
   - This takes 1-2 minutes

3. After that, all requests are fast (2-5 seconds)

4. In browser, you can:
   - Test the /generate endpoint
   - Send prompts
   - See AI responses
   - Try different parameters


YOUR REQUEST EXAMPLE:

   {
     "prompt": "Which is the largest country?",
     "max_new_tokens": 50,
     "temperature": 0.7
   }


EXPECTED RESPONSE:

   {
     "prompt": "Which is the largest country?",
     "generated_text": "Which is the largest country? Russia is the largest country by land area, covering about 17.1 million square kilometers (6.6 million square miles). China is the second largest country, covering about 9.6 million...",
     "status": "success"
   }


QUICK REFERENCE:

API Endpoints:
  GET  http://localhost:8000/              → Health check
  POST http://localhost:8000/generate      → Generate text
  POST http://localhost:8000/chat          → Chat interface

Documentation:
  http://localhost:8000/docs               → Interactive docs (best!)
  http://localhost:8000/redoc              → Alternative docs

Parameters:
  prompt              (string) - Your input text
  max_new_tokens      (int) - How many words to generate (default: 50)
  temperature         (float) - Creativity level 0.1-1.0 (default: 0.7)


FILES YOU NEED:

  main_local.py       ← This is the backend (run this!)
  .env                ← Configuration (API_PORT, HF_MODEL, etc.)
  requirements.txt    ← Dependencies (already installed)


TROUBLESHOOTING:

If you get "Connection refused":
  → Make sure python main_local.py is running
  → The command window should show "Uvicorn running on..."

If port 8000 is already in use:
  → Edit .env and change: API_PORT=8001
  → Then run: python main_local.py again

If you want to stop the server:
  → Press CTRL+C in the command window


NEXT STEPS:

  2. Wait for "Application startup complete"
  3. Open: http://localhost:8000/docs
  4. Try the /generate endpoint
